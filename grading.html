Grading Scale

The final grade will be computed using a weighted average. Two different grading methods will be used in this class. Exams will be graded and a simple percentage will be calculated. Homeworks and discussions will be graded using a 5-point (subject to change) grading scale using the 5 letter grades. Letters will correspond to the top of their range, that is an A is worth 100%, a B is worth 89%. The exception is an F which is worth the floor of 0%.

What is an A?

To get an A on a particular homework or in this class in general, you need to exceed requirements. Meeting requirements is worth a B. Here are some examples of ways you can exceed requirements (assuming these are not explicitly called out in the actual requirements): implement robust error handling, provide written documentation on the use of your project, write tests to prove the correctness of your code, work with a real user to make iterations of your project that implement user feedback, run tests to document the amount of load your code can handle then find areas to improve on that and document the changes you make.
